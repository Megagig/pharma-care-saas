Problem Identified

The Pricing page was throwing an error: useNotifications must be used within a NotificationProvider because the subscription hooks were trying to use notifications outside the provider context.

Root Cause

The useCreateCheckoutSessionMutation and other subscription hooks in useSubscription.ts were calling useNotifications() directly, but the Pricing page is a public route that might be accessed before the NotificationProvider is fully initialized.

Solution Implemented

  • Created useSafeNotifications() hook with fallback handling
  • Replaced all useNotifications() calls with useSafeNotifications()
  • Added graceful fallback to browser alerts when notifications unavailable
  • Maintained full functionality when NotificationProvider is available
  • Fixed all subscription mutation hooks

Code Changes Made

// Before (causing error): const { addNotification } = useNotifications(); // After (safe fallback): const useSafeNotifications = () => { try { return useNotifications(); } catch (error) { return { addNotification: (notification: any) => { console.warn('NotificationProvider not available, using fallback:', notification); if (notification.type === 'error') { alert(`${notification.title}: ${notification.message}`); } } }; } }; const { addNotification } = useSafeNotifications();

Hooks Fixed

  • useCreateCheckoutSessionMutation - Main hook causing the error
  • useCancelSubscriptionMutation - Subscription cancellation
  • useReactivateSubscriptionMutation - Subscription reactivation
  • useUpdateSubscriptionMutation - Subscription updates
  • useAddPaymentMethodMutation - Payment method management
  • useRemovePaymentMethodMutation - Payment method removal
  • useSetDefaultPaymentMethodMutation - Default payment method
  • useDownloadInvoiceMutation - Invoice downloads
