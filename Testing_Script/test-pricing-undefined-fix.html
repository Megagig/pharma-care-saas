Problem Identified

The Pricing page was throwing TypeError: Cannot read properties of undefined (reading 'map') because plan properties like displayedFeatures were undefined when the API returned incomplete data.

Root Cause

The subscription service was returning plan objects with missing or undefined properties, causing the component to crash when trying to access:

  • plan.displayedFeatures.map()
  • plan.priceNGN.toLocaleString()
  • Other plan properties without safe checks

Solution Implemented

  • Added safe array access: (plan.displayedFeatures || []).map()
  • Added safe number access: (plan.priceNGN || 0).toLocaleString()
  • Added safe string access: plan.name || 'Plan'
  • Added safe ID access: plan._id || ''
  • Added conditional rendering for pricing cards section
  • Improved query error handling without mock data
